RANKING + LEADERBOARD SYSTEM
================================================

Goal:
- Create category-based leaderboard (kid/teen/adult).
- Ranking based on Final Score (desc).
- Tie-breakers:
  1) Higher Final Score wins
  2) If tie: shorter readingSeconds wins
  3) If tie: shorter answeringSeconds wins
  4) If still tie: earlier submission time wins

Scoring Rules:
- MCQ: +1 per correct, 0 for wrong
- TEXT: manualScore added by admin
FinalScore = mcqCorrectCount + manualScore

Publishing:
Add Admin control “Publish Results” per category:
- publishedAt per category in DB
- If not published, students cannot see leaderboard or exact results.

Leaderboard UI:
1) Public Leaderboard Page (optional) /leaderboard/:category
- shows Top 100
Columns:
Rank | Name (mask surname like “Mehmet A.” or “Mehmet Asl…”) | Country | City | Score | Reading Time | Answer Time
- Only visible when published

2) Student Dashboard Leaderboard Widget
- Show student’s rank position if published
- Show top 10 preview

Admin Leaderboard Panel
- View full leaderboard per category
- Export CSV
- “Publish / Unpublish” toggle
- “Recalculate leaderboard” button

Recalculate must:
- recompute mcqCorrectCount for all submissions
- ensure finalScore correct
- then order by ranking rules

API ROUTES UPDATES
================================================

Student:
- GET /api/student/leaderboard/:category
  - if resultsPublishedAt is null => return 403 with message
  - else return ordered leaderboard rows
- GET /api/student/my-rank
  - returns user’s rank in their category (only if published)

Admin:
- GET /api/admin/submissions?category=&status=&q=
- GET /api/admin/submissions/:id  // includes answers + question data
- PUT /api/admin/submissions/:id/manual-score
  - update manualScore, finalScore
- PUT /api/admin/submissions/:id/status
- POST /api/admin/leaderboard/recalculate?category=
- PUT /api/admin/results/publish/:category
  - set resultsPublishedAt = now
- PUT /api/admin/results/unpublish/:category
  - set resultsPublishedAt = null
- GET /api/admin/leaderboard/:category
- GET /api/admin/export/leaderboard.csv?category=

Computation Requirements (server-side):
- On finish-competition:
   - compute mcqTotalCount
   - compute mcqCorrectCount by comparing answers to correctAnswer
   - set autoScore = mcqCorrectCount
   - set finalScore = autoScore + manualScore (manualScore default 0)
   - set status SUBMITTED
- On manual scoring update:
   - recompute finalScore

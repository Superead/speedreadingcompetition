Upgrade the platform from single-competition mode to MULTI-COMPETITION architecture.

CORE CONCEPT
The system must support unlimited competitions.
Each competition belongs to ONE category (kid, teen, adult).
A student can participate in each competition at most ONCE.

=================================================

=================================================
RULES
=================================================

1. Student can register only ONCE per competition.
2. Student can participate in multiple competitions over time.
3. Student dashboard must list AVAILABLE competitions for their category.
4. Student dashboard must list COMPLETED competitions for that student.
5. Admin can create, edit, publish and close competitions independently.

=================================================
STUDENT FLOW (UPDATED)
=================================================

Student Dashboard:

Section: Available Competitions
- List all competitions:
   WHERE category = student.category
   AND now BETWEEN registrationStartTime AND registrationEndTime
   AND NOT EXISTS CompetitionRegistration for this student

Section: My Competitions
- List competitions where student is already registered
- Show status: Registered | In Progress | Submitted | Closed
- Show buttons based on phase

Registration Button:
- Registers into CompetitionRegistration
- Prevent duplicate registration

Competition Flow:
- All reading, answering, submissions tied to competitionId

=================================================
ADMIN FLOW (UPDATED)
=================================================

Admin Dashboard:

Competition Manager:
- Create competition
- Edit competition
- Assign category
- Set all time windows
- Assign book to competition
- Assign questions to competition
- Publish / Close competition

Submissions & Leaderboard:
- Filter by competitionId
- Each competition has its own leaderboard
